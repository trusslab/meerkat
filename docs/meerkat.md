# Using Meerkat (./bin/meerkat)

This documentation is useful if you wish to set up Meerkat to run in an active environment rather than a pre-parsed dataset.

## Setup

This is the setup for an individual bug. For general setup, see [docs/README.md](README.md).

The data for each bug should be placed in specific locations in it's working directory.
For that, I use the scheme `wd-meerkat-#` corresponding to Meerkat's ID parameter.
Begin with the following directories:
```
wd-meerkat-1/
├── bugs
|   ├── primary
│   └── alias#
└── reproducers
```
There can be any number of aliases as Meerkat will treat them all as the same bug.
This can be useful if inter-report deduplication has already been performed.

Place all known reproducers in `wd/reproducers` and for each bug alias, provide the bug name in `wd/bugs/alias/description` and a `report` when possible.
One of the alias needs to be named 'primary' for the purpose of naming the log.
The `description` files are the same ones from Syzbot.
They should be a single line with just the bug name.
The `report` files are also the same ones generated by Syzbot.

Next, download the kernel config (`kernel.cfg`). This can be placed anywhere, but in the working directory is the easiest to keep track of.
```
wd-meerkat-1/
├── bugs
│   └── primary
│       ├── description
│       └── report
├── kernel.cfg
└── reproducers
    ├── repro-1.prog
    ├── repro-2.prog
    └── repro-3.prog
```

Next we write the configuration file for Meerkat (`meerkat.cfg`).
It is read as a json, but it's my own library, so it may not support everything.
Keep it simple.
```
{
    "bugID": "bug0001",
    "bug_name": "WARNING in ieee80211_link_info_change_notify",
    "aliases": "/path/to/wd-meerkat-1/bugs/",
    "bug_link": "https://syzkaller.appspot.com/bug?id=64a91d95c31faa0e8dd58a5a7cc0ab963e28600a",
    "anchor_hash": "b229b6ca5abbd63ff40c1396095b1b36b18139c3",
    "arch": "amd64",
    "repository": "https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git",
    "branch": "master",
    "kernel_config": "/path/to/kernel.cfg",
    "reproducers": "/path/to/reproducers/",
    "reproducer": "repro-1.prog",
    "wd": "/path/to/wd-meerkat-1/",
    "home": "/path/to/meerkat/",
    "syzkaller": "/path/to/meerkat/syzkaller/",
    "compilers": "/path/to/meerkat/compilers/",
    "compiler": "gcc",
    "ccache": "ccache",
    "image": "/path/to/stretch.img",
    "image_key": "/path/to/stretch.id_rsa"
}
```
`bugID`, `bug_name`, `bug_link`, `anchor_hash`, `branch`, `reproducer`, and `ccache` are not required.
In fact, they often get overwritten by other user-provided options.
(Look, this is usually all automatically generated and read. I didn't really care if there was extra stuff in it.)
Put this config in the working directory as well just for safe keeping.

Finally, just run Meerkat:
```
./bin/meerkat -i 1 -c path/to/meerkat.cfg -a <anchor hash> -F features 2>&1 | tee wd/log/bug0001.log
```
